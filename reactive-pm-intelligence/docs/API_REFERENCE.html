Authentication Model

Secure Enough For Pilot, Structured For Enterprise Progression

  • `POST /auth/token` returns the pilot bearer token.
  • Protected routes require `Authorization: Bearer <token>`.
  • Health, metrics, and mock integration endpoints remain open for operational visibility.
  • The OpenAPI contract already models `bearerAuth` so the path to enterprise identity is clean.

Primary Schema Families

  • Request schemas: `IngestRequest`, `AssistantRequest`.
  • Analytics schemas: `PatternResponse`, `PredictionResponse`, `PMScheduleResponse`.
  • Executive schemas: `KPIResponse`, `PortfolioOverviewResponse`, `AssistantResponse`.
  • Operational schemas: `HealthResponse`, `PipelineRunResponse`, `CountsResponse`.
POST/auth/token

Issues the pilot bearer token used across protected routes.

POST/pipeline/run

Rebuilds recurring patterns, predictions, and PM schedules from current history.

GET/portfolio/overview

Returns portfolio scale, hotspot facilities, and at-risk assets for leadership and planners.

GET/dashboard/kpis

Returns the KPI payload already shaped for the live dashboard.

POST/assistant/chat

Exposes the PM Copilot interface using live portfolio context and mock or LLM-backed response generation.

POST/pm/generate

Converts high-risk recurrence predictions into PM schedules ready for downstream operational action.
